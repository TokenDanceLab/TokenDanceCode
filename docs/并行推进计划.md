# TokenDanceCode 并行推进计划

更新时间：2026-06-09

本文档是公开维护计划。它记录当前稳定基线、已采用的架构决策、下一轮工作队列和合并顺序。内部 agent 名称、agent ID、本机 worktree 路径和私有 prompt 不写入这里。

## 稳定基线

当前主线是 `codex/ts-refactor` 的 TypeScript monorepo，候选版本为 `0.2.0-ts.0`。`release/npm-first` 指向 npm first-candidate 系列提交；npm registry 仍未公开显示 `@tokendance/code-core`、`@tokendance/code-sdk`、`@tokendance/code-cli`。

最近稳定 gate：

- `pnpm contract:check`
- `pnpm verify`
- `pnpm pack:smoke`
- `pnpm registry:next:check`
- `pnpm release:next:check`
- `git diff --check`

测试数量以命令输出为准，不在本文档固化。

## 架构决策

TokenDanceCode 对标 Claude Code、Codex CLI 和 OpenCode，但只吸收适合自用本地 Agent 框架的部分。

保留：

- 薄 CLI，复杂行为下沉到 `packages/core` 和 `packages/sdk`。
- command registry lane，避免 help、usage、别名、JSON contract 和 handler 漂移。
- OpenCode command metadata registry 的思路，限于 id、category、title、aliases、usage 和 JSON contract。
- Codex contract/schema drift gate，保护 SDK manifest、AgentHub event envelope、transcript schema、provider schema 和 CLI JSON 输出。
- provider-neutral runtime，覆盖 OpenAI Responses API、OpenAI Chat Completions / TokenDance Gateway、Anthropic-compatible Messages。
- 权限 pipeline、transcript/resume、context/compact、task/todo、worktree/subagent 的本地实现。
- AgentHub 可消费 SDK surface，包括 event sink、approval bridge、doctor/config、TokenDanceID OIDC login URL helper 和 package feature flags。

拒绝：

- 拒绝 app-server daemon。
- 拒绝 OpenTUI 或复杂 full-screen TUI。
- 拒绝 plugin marketplace。
- 拒绝 native installer。
- 拒绝把 TokenDanceCode 描述为 AgentHub 替代品、团队协作 SaaS、IDE 插件或云端平台。

## 当前风险

- CLI main.ts 过大风险：新增命令必须优先使用 command metadata registry 和 helper 分层，避免继续堆在入口文件。
- SDK/AgentHub contract 漂移：AgentHub 消费方应以 `TOKEN_DANCE_CODE_PACKAGE.agentHub.agentStreamSchemaVersion` 和 feature flags 做启动检查。
- npm first publish 风险：发布只能由 release owner 手动执行，检查脚本不能调用 `npm publish`。
- 真实 provider smoke 风险：只允许显式 opt-in，本地 key 不进入仓库、README、测试 fixture 或公开 release docs。

## Merge Order

Wave 7 建议合并顺序：

1. 本地交互式审批闭环。
2. `run --json` / `--stream-json`，集中处理 CLI command/main 冲突。
3. 路径/命令级权限策略。
4. Context budget 与可恢复 compact。
5. 确定性本地 hooks。
6. Same-session run concurrency policy。

每个切片合并前做三件事：

1. Review worker diff，只保留目标文件边界内的改动。
2. 重跑对应 focused tests 和 `pnpm typecheck`。
3. 更新 [并行执行状态.md](并行执行状态.md) 为公开进度日志，只写用户可见变化、提交和验证命令。

整轮合并后运行：

```powershell
pnpm release:next:check
git diff --check
```
