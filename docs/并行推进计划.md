## 并行原则

- 每个方向使用独立 worktree 和独立 branch，避免互相覆盖。
- 每个 worktree 必须以 TDD 推进：先加失败测试，再实现，再补文档。
- 每个 worktree 的完成定义都是 `pnpm verify` 通过；涉及包内容时还要跑 `pnpm pack:check`。
- TokenDance Gateway 模型调用继续使用 TokenDance API key；TokenDanceID/OIDC 只作为 AgentHub/Hub 或管理界面的身份会话平面。
- 不发布 npm、不碰生产、不写真实 secret。npm 只做发布准备和本地 pack/install smoke，发布需要单独确认。
- TUI 可以新增 UI 包或 CLI 子命令，但不得把当前薄 CLI 变成混乱的大文件；复杂渲染组件应拆出独立模块。
