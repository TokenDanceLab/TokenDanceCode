## 本轮已完成

### npm 发布候选基线

- `release/npm-first` 已跟随 `codex/ts-refactor` 推送到 GitHub。
- `pnpm release:publish:check` 已作为最终本地发布预检：
  - 要求 clean worktree。
  - 要求 `HEAD` 与本地/远端 `release/npm-first` 一致。
  - 串行运行 registry、contract、verify、build、dry-run pack、reviewed tarball smoke。
  - 每个 gate 失败后立即停止，不继续生成 tarball 或打印 publish 命令。
  - 只打印人工发布命令，不执行 `npm publish`。

### CI 和发布安全

- 新增 GitHub Actions workflow：`.github/workflows/ci.yml`。
- CI 在 `windows-latest` 上运行：
  - `pnpm install --frozen-lockfile`
  - `pnpm contract:check`
  - `pnpm verify`
  - `pnpm pack:smoke`
  - `pnpm registry:next:check`
- CI 不配置 npm token，不运行 npm publish。
- `pack:smoke` 已强化 tarball privacy scan：
  - 扫描真实 `.tgz` 条目名。
  - 解包后扫描所有小文件内容。
  - 安装后继续扫描 `node_modules/@tokendance`。
  - 阻止 `.npmrc`、`.env`、private key 文件、本机路径、npm/GitHub/provider token 形态进入发布包。

### CLI / SDK / OIDC

- CLI 支持 direct prompt：`tokendance "hello"` 等价走单轮 run 逻辑。
- SDK `doctor()` 已包含 `TOKENDANCE_GATEWAY_API_KEY` present/missing 状态，方便 AgentHub 启动检查。
- TokenDanceID OIDC helper 已要求自定义 scope 必须包含 `openid`。
- CLI 的 `auth tokendanceid login-url --scope "profile email"` 会失败，不输出 authorize URL。

### 文档公开化

- README 中文为主，英文 README 保持同步入口。
- README 增加 npm 发布后的安装路径：
  - `npm install -g @tokendance/code-cli@next`
  - `npm install @tokendance/code-sdk@next @tokendance/code-core@next`
- 公开文档已清理本机目录拓扑、内部 secret-store 说法和旧 Python-first 口径。
- 旧 Python 实现只保留为迁移参考，不再作为当前验收主线。

- `tokendance sessions --json`

### P1：CLI main 拆分

`packages/cli/src/main.ts` 已经偏大。后续新增功能应先抽 helper，再接线：

- session/transcript commands
- diagnostics/config/gateway commands
- work/task/todo commands
- auth commands

目标是让 `main.ts` 保持为 IO wiring，不继续承载业务逻辑。

### P1：AgentHub npm consumer example

保持 `@tokendance/code-agenthub-example` 私有，不发布。

### P2：Provider smoke 扩展

当前真实 provider smoke 主要覆盖 TokenDance Gateway。后续可补：
