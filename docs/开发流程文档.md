# TokenDanceCode 开发流程文档

日期：2026-06-09

本文档描述当前 TypeScript monorepo 的开发循环。旧 Python 版本只保留为迁移参考，新功能不再写入 `src/tokendance` 或旧 `tests/` 目录。

## 开发原则

每个变更同时处理代码、测试和文档：

- 代码：按职责写入 `packages/core`、`packages/sdk`、`packages/cli` 或 `packages/agenthub-example`。
- 测试：公共行为必须有 Vitest 覆盖，优先覆盖 runtime event、permission decision、provider mapping、transcript/resume、SDK facade 和 CLI JSON 输出。
- 文档：用户可见行为更新 README 或 package README；架构和发布边界更新 `docs/` 下对应文档。

搜索和定位默认使用 `rg`：

```powershell
rg --files
rg -n "AgentHub|permission|run.agent.result" packages docs
```

## 仓库结构

```text
TokenDanceCode/
  packages/
    core/              runtime, providers, tools, permissions, sessions
    sdk/               AgentHub and script-facing public SDK
    cli/               tokendance command line entrypoint
    agenthub-example/  private integration fixture for AgentHub-style consumers

  README.md            Chinese-first public entry

```

The Python tree remains in the repository for v0.1 comparison only. Do not extend it when adding TypeScript behavior.

## Package Ownership

| Area | Owner Package | Typical Tests |
|---|---|---|
| Provider adapters, tools, permissions, transcript, context, compact/resume | `packages/core` | `packages/core/tests/*.test.ts` |
| AgentHub facade, event mapping, approval bridge, TokenDanceID helper, package metadata | `packages/sdk` | `packages/sdk/tests/*.test.ts` |
| Command parsing, help, renderer, local interactive flow | `packages/cli` | `packages/cli/tests/*.test.ts` |
| Copyable AgentHub consumer fixture | `packages/agenthub-example` | `packages/agenthub-example/tests/*.test.ts` |

When a change crosses package boundaries, keep the lower-level contract in `core`, expose the stable surface from `sdk`, and keep CLI orchestration thin.

## Feature Workflow

1. Read README, AGENTS, and the owning doc for the area.
2. Use `rg` to locate existing implementation and tests.
3. Add or adjust a failing focused test.
4. Implement the smallest behavior that satisfies the test.
5. Run the focused test and `pnpm typecheck` when TypeScript surfaces changed.
6. Update README/package README/docs when behavior, commands, environment variables, or release gates changed.
7. Run `git diff --check`.

For larger independent slices, use repo-local worktrees and keep each lane to a disjoint file boundary. The coordinator owns conflict resolution and final release-gate verification.

## Verification Levels

Focused development:

```powershell
pnpm test <test files>
pnpm typecheck
git diff --check
```

Package and SDK contract changes:

```powershell
pnpm contract:check
pnpm pack:smoke
```

Release review candidate:

```powershell
pnpm registry:next:check
pnpm release:next:check
```

`npm publish` is not a verification command. It remains a separate release-owner action after package contents, npm org access, dist-tag, and license intent are reviewed.

## Provider And Secret Rules

TokenDanceCode supports OpenAI Responses, OpenAI Chat Completions compatible endpoints, TokenDance Gateway, and Anthropic-compatible Messages. Configuration precedence belongs in `packages/core/src/config.ts` and must stay covered by config/provider tests.

Secrets are never committed. Public docs may name environment variables such as `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `TOKENDANCE_GATEWAY_API_KEY`, and `TOKENDANCE_GATEWAY_BASE_URL`, but examples must use placeholders. Real provider smoke tests require explicit opt-in from a controlled local shell or ignored operator environment.

## Current Active Queue

The active TS baseline has:

- pnpm workspace and TypeScript project references.
- `@tokendance/code-core`, `@tokendance/code-sdk`, and `@tokendance/code-cli`.
- Mock provider, tool loop, permission events, JSONL transcript store, SDK `run` and `runStreamed`.
- OpenAI Responses, OpenAI-compatible Chat/TokenDance Gateway, and Anthropic-compatible provider adapters.
- AgentHub SDK facade, event sink mapping, remote approval bridge, TokenDanceID OIDC login URL helper, doctor/config facade.
- npm candidate gates: `contract:check`, `release:next:check`, tarball install smoke, privacy scan, and manual publish boundary.
- AgentHub terminal failure mapping: provider/runtime failure emits `turn.failed` and maps to `run.agent.result` with `success=false`.
- Context budget / compact recovery: `ContextBuilder` supports instruction, compact summary, memory, and recent-message character budgets; manual compact writes recovery notes and recent transcript snippets back to session state.
