## 并行原则

- 每个方向使用独立 worktree 和独立 branch，避免互相覆盖。
- 每个 worktree 必须以 TDD 推进：先加失败测试，再实现，再补文档。
- 每个 worktree 的完成定义都是 `pnpm verify` 通过；涉及包内容时还要跑 `pnpm pack:check`。
- TokenDance Gateway 模型调用继续使用 TokenDance API key；TokenDanceID/OIDC 只作为 AgentHub/Hub 或管理界面的身份会话平面。
- 不发布 npm、不碰生产、不写真实 secret。npm 只做发布准备和本地 pack/install smoke，发布需要单独确认。
- TUI 可以新增 UI 包或 CLI 子命令，但不得把当前薄 CLI 变成混乱的大文件；复杂渲染组件应拆出独立模块。

## GPT-5.5 High Agent Prompts

每个并行 agent 使用同样的硬约束：

```text
You are working in the assigned TokenDanceCode worktree only.
Base branch is codex/ts-refactor at the documented baseline.
Do not modify sibling worktrees. Do not publish npm. Do not touch production or secrets.
Use TDD: add failing tests first, observe failure, implement, then update docs.
Keep changes scoped to the assigned workstream. Run the listed focused tests and pnpm verify.
Commit only your worktree's scoped changes with a clear message.
Report: changed files, tests, residual risks, and merge notes.
```

Workstream-specific prompts should include the matching row from the matrix, the relevant files, and the acceptance commands.

## Merge Strategy

1. Review each worktree independently: `git status`, focused tests, `pnpm verify`, and diff review.
2. Merge in this order to reduce conflicts:
   - `W3-LLM-Providers`
   - `W4-Safety-Permissions`
   - `W5-Threads-Tasks`
   - `W2-SDK-AgentHub`
   - `W1-CLI-TUI`
   - `W6-Brand-Release-Tests`
3. After each merge, run `pnpm verify`.
4. Before npm prerelease, run `pnpm pack:check`, local tarball install smoke, and package metadata review.
