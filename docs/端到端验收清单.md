# 端到端验收清单

本文档对应 `docs/开发流程文档.md` 的阶段 15：安装、文档与端到端验收。手动验收建议在一次性测试仓库或独立 worktree 中执行，避免污染正在开发的主工作区。

> TS 重构分支更新：当前权威验收先看本节。下面 Python 验收项保留为 v0.1 参考，直到对应功能迁移到 TypeScript 后再逐项替换。

## TS-0. 当前 TypeScript 闭环验收

- [x] 在 `codex/ts-refactor` worktree 中安装依赖。

```powershell
pnpm install
```

- [x] 运行基础验证。

```powershell
pnpm verify
pnpm contract:check
pnpm pack:check
pnpm pack:smoke
pnpm release:next:check
```

当前证据以最近一次 `pnpm release:next:check` 输出为准；静态文档不再硬编码测试数量，避免开发中间态误导验收结论。`contract:check` 只读检查 SDK package manifest、AgentHub event/fixture readiness、pack smoke 入口和 npm publish 禁止边界；`pack:check` 会构建、dry-run 打包 core/sdk/cli，并运行本地 tarball install smoke；`release:next:check` 覆盖 `pnpm contract:check && pnpm verify && pnpm pack:check`。

- [x] 验收 npm `next` 预发布包基线。

```powershell

pnpm release:next:check
pnpm pack:smoke
```

期望：public 包仅包括 `@tokendance/code-core`、`@tokendance/code-sdk`、`@tokendance/code-cli`；三个包 manifest 都包含 MIT license、repository directory、homepage、bugs、keywords、`publishConfig.access=public`、`publishConfig.tag=next` 和 package-local README；根 LICENSE 存在；只读 `pnpm contract:check` 覆盖 SDK package manifest、AgentHub event/fixture readiness、pack smoke 入口和 npm publish 禁止边界；SDK manifest 暴露 `pnpm verify`、`pnpm pack:check`、`pnpm pack:smoke`、`pnpm release:next:check`。`pnpm pack:smoke` 应在临时项目中安装真实 tarball，并验证 SDK import、mock turn、`tokendance --version`、`tokendance doctor --json` 的 AgentHub readiness / `provider-ready`，以及 `tokendance quality --json` 的结构化结果。不要在检查脚本中执行 npm publish；`npm publish --tag next` 只作为 release owner 审核后的人工发布动作。
